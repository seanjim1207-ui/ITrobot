## 部署流程

### 開發環境
1. 確認所有測試通過：`npm test`
2. 建置專案：`npm run build`
3. 本地預覽：`npm run preview`

### 正式環境部署
1. 合併 PR 到 `main` 分支
2. CI/CD 會自動觸發部署
3. 確認部署狀態：查看 GitHub Actions

### 回滾步驟
如果部署後發現問題：
1. 到 GitHub Actions 找到上一次成功的部署
2. 點擊 Re-run 重新執行
3. 通知團隊成員

### 注意事項
- 部署前務必確認環境變數已設定
- 大版本更新需先通知使用者


### C槽滿了要刪什麼

**症狀:**  同事會問C槽滿了，但不知道刪什麼

**解決方法:**
1.可以先到此路徑:"S:\檢測軟體\硬碟空間\SpaceSniffer"
2.滑鼠右鍵"已系統管理員身分執行"
3.開資料夾找到相對路徑手動刪除
4.通常.cache命名可以刪除

**常見問題:**
1.可以到 C:\Users\使用者名稱
2.點選【檢視】→隱藏的項目→會顯示隱藏資料夾"AppData"
3.點選後點開→local→Temp ，裡面的檔案都可以清空
4.某些同事下班電腦會被抓去當算圖機，如果有moonreder帳戶的temp很多也可以刪除

### F槽不見了
**症狀:**  員工正常登入後，預設會出現F槽，但沒有出現

**解決方法:**
1.電腦帳號登出登入後，F槽就會出現。
2.滑鼠右鍵點選】→左下角的開始列→【關機或登出】→登出

### 新進人員沒有F槽
**症狀:**  員工登入後，預設會出現F槽，但沒有出現

**解決方法:**
1.先打開 Active Directory 管理介面，找左側【 Employee】裡的群組
2.在群組中找到員工的帳號，點選使用者會跳出設視窗，選擇【設定檔】
3.找到下方的【主資料夾】→點選【連線】
4.輸入:\\userdata\userdata\user\使用者帳號

### F槽滿了請幫我清空
**症狀:**  同事在F槽內刪除掉檔案後，硬碟空間並沒有變多

**解決方法:**
1.原因是F槽的檔案刪除後，會到NAS的垃圾桶，必須清空NAS的垃圾桶
2.下載軟體"putty"，用SSH方式我們將連到主機
3.在【Host Name (or IP address)】格子內輸入: "userdata"
3.出現遠端小黑窗時，帳號:"root" 密碼:"at-ha11@storage"
4.```資源回收桶位置:/mnt/backup/.recycle/```
5.先開啟mis.moonshine.tw查詢同事帳號，再輸入"ls"確認同事資料夾
![[Pasted image 20260409102820.png]]
6.```刪除指令: rm -rf /mnt/backup/.recycle/同事帳號```

### 網路位置S槽不見
**症狀:**  同事反映S槽 或 某槽沒有出現

**解決方法:**
1.開一個資料夾，路徑輸入:``` \\qsan\Software\_RD\BAT```
2.點兩下 【addCmdKeyFoS.bat】就會出現網路磁碟S槽
3.以此類推其他槽，ex: U槽不見點【addCmdKeyFoU.bat】

### 在U槽沒看到某資料夾
**症狀:**  連上U槽後，但同事反映U槽沒看到一些資料夾(ex:)，常發生在新進同事

**解決方法:**
0.原因是某些槽有隱藏資料夾有鎖權限，要到AD中給予
1.先打開 Active Directory 管理介面，找左側【 Employee】裡的群組
2.在群組中找到員工的帳號，點選使用者會跳出設視窗，選擇【 隸屬於】
3.點選【 新增】→ 在"輸入物件名稱來選取"的區域中輸入: "grp" →在按下【 檢查名稱】 →選擇 "Grp_fulltime" → 確定 。
 4.以此類推，詳細的資料夾權限請詢問主管

### 新進PM/PA/PC管理職沒有M槽
**症狀:**  有管理職的同事點M槽的bat後，網路磁碟是鎖著的

**解決方法:**
1.先打開 Active Directory 管理介面，找左側【 Employee】裡的群組。
2.找到員工的帳號，將使用者拉到【 Employee】群組下的【Management】群組。
3.點選使用者會跳出設視窗，選擇【 隸屬於】。
4.點選【 新增】→ 在"輸入物件名稱來選取"的區域中輸入: "grp" 
 → 在按下【 檢查名稱】 →選擇 "Grp_Temp_M管理組" → 確定 。 
 5.請同事將電腦登出登入。
6.M槽多為管理職資料，可以多注意新進員工身分是否需要適合。
